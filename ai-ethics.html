Етичні принципи AI

Благодійність (Beneficence)

AI повинен приносити користь людству та сприяти добробуту суспільства

Приклади застосування:

  • Медичні AI-системи для діагностики захворювань
  • AI для боротьби зі зміною клімату
  • Освітні AI-платформи для доступної освіти
  • AI для підвищення безпеки на дорогах

У робочому середовищі:

Використовуйте AI для покращення умов праці
Автоматизуйте рутинні завдання, щоб звільнити час для творчості
Використовуйте AI для навчання та розвитку команди

Не нашкодь (Non-maleficence)

AI не повинен завдавати шкоди людям або суспільству

Потенційні ризики:

Фізичні ризики
  • Неправильна діагностика в медицині
  • Аварії автономних транспортних засобів
  • Збої в системах безпеки
Психологічні ризики
  • Залежність від AI-систем
  • Втрата навичок критичного мислення
  • Соціальна ізоляція
Соціальні ризики
  • Масове безробіття
  • Поглиблення нерівності
  • Дискримінація та упередженість

Високий контроль: Медичні діагнози, судові рішення

Упередженість та справедливість

Що таке алгоритмічна упередженість?

Систематичні та несправедливі переваги або дискримінація певних груп людей в AI-системах, що призводить до нерівних результатів.

Джерела упередженості:

Дані

Проблема: Неповні, неточні або упереджені тренувальні дані

Приклад: Система розпізнавання облич, тренована переважно на фото білих чоловіків
Наслідок: Погана точність для жінок та людей інших рас

Алгоритм

Проблема: Неправильний вибір моделі або параметрів

Приклад: Алгоритм кредитного скорингу, що надає перевагу певним поштовим кодам
Наслідок: Дискримінація за географічним принципом

Людський фактор

Проблема: Неусвідомлені упередження розробників

Приклад: Система підбору персоналу, що відображає упередження HR-спеціалістів
Наслідок: Дискримінація жінок у технічних професіях

Контекст використання

Проблема: Використання AI в неналежному контексті

Приклад: Модель, тренована на одній популяції, використовується для іншої
Наслідок: Неточні результати для нової популяції

Типи упередженості в AI:

Статистична упередженість

Різна точність моделі для різних груп

Приклад: Система медичної діагностики з точністю 95% для чоловіків та 80% для жінок
Як виявити: Аналіз метрик точності по групах

Упередженість рівності можливостей

Різні шанси на позитивний результат для кваліфікованих кандидатів

Приклад: Кваліфіковані жінки мають менше шансів пройти AI-скринінг резюме
Як виявити: Порівняння True Positive Rate між групами

Демографічна паритетність

Різні показники позитивних рішень для різних груп

Приклад: 30% чоловіків отримують схвалення кредиту vs 20% жінок
Як виявити: Порівняння загальних показників схвалення

Стратегії зменшення упередженості:

На етапі підготовки даних

Збалансування датасету
Забезпечення репрезентативності всіх груп
Інструменти: SMOTE, ADASYN, підвибірка
Аугментація даних
Штучне збільшення недопредставлених груп
Інструменти: GANs, синтетичні дані
Видалення чутливих атрибутів
Виключення прямих ознак дискримінації
Обережно: можуть залишитися проксі-змінні

На етапі навчання моделі

Справедливі алгоритми
Використання алгоритмів з вбудованими обмеженнями справедливості
Fairlearn, AIF360, Themis
Мульти-об'єктивна оптимізація
Балансування між точністю та справедливістю
Парето-оптимізація, зважені функції втрат
Адверсаріальне навчання
Навчання моделі ігнорувати чутливі атрибути
Adversarial debiasing

На етапі пост-обробки

Калібрування результатів
Коригування порогів для різних груп
Threshold optimization
Перерозподіл результатів
Зміна фінальних рішень для досягнення справедливості
Outcome redistribution

Метрики справедливості:

Demographic Parity

P(Ŷ=1|A=0) = P(Ŷ=1|A=1)

Однакова ймовірність позитивного результату для всіх груп

Equalized Odds

TPR₀ = TPR₁ та FPR₀ = FPR₁

Однакові показники помилок для всіх груп

Calibration

P(Y=1|Ŷ=s,A=0) = P(Y=1|Ŷ=s,A=1)

Однакова точність прогнозів для всіх груп

Приватність та безпека

Принципи захисту приватності:

Мінімізація даних

Збирайте лише необхідні дані для конкретної мети

Приклад: Для рекомендації фільмів не потрібні медичні дані

Цільове використання

Використовуйте дані лише для заявленої мети

Приклад: Дані для медичної діагностики не можна використовувати для страхування

Обмеження зберігання

Зберігайте дані лише стільки, скільки необхідно

Приклад: Автоматичне видалення персональних даних через 2 роки

Контроль користувача

Надайте користувачам контроль над їх даними

Приклад: Можливість переглянути, виправити або видалити дані

Технології захисту приватності:

Диференційна приватність

Додавання контрольованого шуму для захисту індивідуальних записів

Як працює: Алгоритм додає випадковий шум до результатів запитів
Переваги: Математичні гарантії приватності
Використання: Apple (iOS analytics), Google (Chrome telemetry)
Приклад:

Замість точної кількості користувачів (1,234,567) повертається приблизне значення (1,235,000 ± шум)

Федеративне навчання

Навчання моделей без централізованого збору даних

Як працює: Модель навчається локально на пристроях, обмінюючись лише оновленнями
Переваги: Дані залишаються на пристроях користувачів
Використання: Google Keyboard, Apple Siri
Процес федеративного навчання:
1
Сервер відправляє початкову модель
2
Пристрої навчають модель локально
3
Пристрої відправляють оновлення
4
Сервер агрегує оновлення

Гомоморфне шифрування

Обчислення на зашифрованих даних без їх розшифрування

Як працює: Математичні операції виконуються на зашифрованих даних
Переваги: Дані ніколи не розшифровуються під час обробки
Обмеження: Повільні обчислення, обмежений набір операцій

Синтетичні дані

Генерація штучних даних, що зберігають статистичні властивості оригіналу

Як працює: AI генерує нові дані на основі паттернів з реальних даних
Переваги: Немає ризику розкриття особистої інформації
Використання: Тестування, дослідження, навчання моделей

Загрози безпеки AI-систем:

Adversarial Attacks

Спеціально створені вхідні дані, що обманюють AI-модель

Приклад: Невидимі для людини зміни в зображенні, що змушують AI класифікувати кота як собаку
Вплив: Неправильні рішення в критичних системах (медицина, безпека)
Захист:
  • Adversarial training
  • Input preprocessing
  • Ensemble methods
  • Certified defenses

Model Inversion

Відновлення тренувальних даних з навченої моделі

Приклад: Відновлення облич людей з моделі розпізнавання
Вплив: Порушення приватності учасників тренувального датасету
Захист:
  • Диференційна приватність
  • Обмеження доступу до моделі
  • Регуляризація

Data Poisoning

Внесення шкідливих даних у тренувальний датасет

Приклад: Додавання неправильно позначених зображень для зміни поведінки моделі
Вплив: Модель навчається неправильним паттернам
Захист:
  • Валідація даних
  • Outlier detection
  • Robust training methods
  • Data provenance tracking

Membership Inference

Визначення, чи був конкретний запис у тренувальних даних

Приклад: Визначення, чи була особа в медичному датасеті
Вплив: Розкриття чутливої інформації про участь у дослідженні
Захист:
  • Диференційна приватність
  • Regularization
  • Early stopping

Довіра користувачів

Люди більше довіряють системам, які можуть пояснити свої рішення

Виявлення помилок

Пояснення допомагають знайти та виправити проблеми в моделях

Етична відповідальність

Можливість перевірити справедливість та відсутність упередженості

Спектр пояснюваності:

Висока пояснюваність
Лінійна регресія, Дерева рішень
Прості для розуміння, але обмежені можливості
Середня пояснюваність

Баланс між точністю та інтерпретованістю
Низька пояснюваність
Нейронні мережі, Deep Learning
Висока точність, але "чорна скринька"

Методи пояснення AI:

Локальні пояснення

Пояснюють конкретне рішення для одного прикладу

LIME (Local Interpretable Model-agnostic Explanations)

Як працює: Навчає просту модель локально навколо конкретного прикладу

Приклад: "Ця email класифікована як спам через слова 'безкоштовно' та 'терміново'"
Переваги: Працює з будь-якою моделлю, зрозуміло для людей
Недоліки: Може бути нестабільним, лише локальні пояснення
SHAP (SHapley Additive exPlanations)

Як працює: Використовує теорію ігор для розподілу "внеску" кожної ознаки

Приклад: "Вік (+0.3), Дохід (+0.2), Кредитна історія (-0.1) → Схвалення кредиту"
Переваги: Математично обґрунтований, стабільний
Недоліки: Обчислювально складний для великих моделей
Grad-CAM

Як працює: Візуалізує, на які частини зображення "дивиться" нейронна мережа

Приклад: Підсвічування області з пухлиною на медичному знімку
Переваги: Інтуїтивно зрозуміло для зображень
Недоліки: Тільки для CNN, може бути неточним

Глобальні пояснення

Пояснюють загальну поведінку моделі

Feature Importance

Як працює: Ранжує ознаки за їх важливістю для моделі

Приклад: "Найважливіші фактори для прогнозу ціни: локація (40%), площа (25%), рік будівництва (15%)"
Partial Dependence Plots

Як працює: Показує, як зміна однієї ознаки впливає на прогноз

Приклад: Графік залежності ймовірності схвалення кредиту від доходу
Surrogate Models

Як працює: Навчає просту інтерпретовану модель імітувати складну

Приклад: Дерево рішень, що апроксимує нейронну мережу

Контрфактичні пояснення

Показують, що потрібно змінити для іншого результату

Counterfactual Explanations

Як працює: Знаходить мінімальні зміни для зміни рішення

Приклад: "Якби ваш дохід був на $5000 більше, кредит було б схвалено"
Переваги: Дає практичні поради для покращення результату

Принципи відповідального AI:

Людино-центричність

AI повинен доповнювати та покращувати людські здібності, а не замінювати людей повністю

Як реалізувати:
  • Залишайте критичні рішення за людьми
  • Надавайте можливість перевизначити AI-рішення
